# 2. 스프링 컨테이너 도입, 싱글톤 레지스트리

## 1. 목적

- 순수 Java 기반에서 DI 컨테이너(`AppConfig`)가 가진 다중 요청 시의 '객체 중복 생성' 문제 해결.
- 스프링 프레임워크의 제어의 역전(IoC).
- 역할과 구현의 명확한 분리를 통한 설정 클래스 구조 개선.

## 2. 구현 내용

- Spring 도입, `@Configuration` 과 `@Bean` 애노테이션 적용 및 빈 생성 로직 정리

```java

@Configuration // 애플리케이션의 설정 정보로 사용되는 클래스임을 표시한다
public class AppConfig {

    /*
     일반적인 객체 생성과 다를 바 없다.
     다만 스프링 IoC 컨테이너가 생성, 의존성 주입, 생명 주기를 관리하는 자바 객체 인 것.
     */
    @Bean
    public MemberService memberService() {
        return new MemberServiceImpl(memberRepository());
    }

```
