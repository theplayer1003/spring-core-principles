# 1. 스프링 코어 및 객체 지향

- 애노테이션 (Annotation, `@`): 소스 코드에 메타데이터를 삽입하여, 컴파일러나 런타임 프레임워크가 특정 동작을 수행하도록 지시하는 식별자
- IoC (Inversion of Control, 제어의 역전): 프로그램의 제어 흐름인 객체 생성, 조립, 소멸을 개발자가 아닌 외부 프레임워크가 주도하는 설계 패러다임
- DI (Dependency Injection, 의존성 주입): 객체가 필요로 하는 외부 부품(의존성)을 내부에서 직접 생성하지 않고, 외부 컨테이너가 런타임에 조립하여 넣어주는 디자인 패턴
- 스프링 컨테이너 (Spring Container): 애플리케이션 구동 시 객체의 생명주기를 전담하여 관리하고, DI를 수행하는 런타임 메모리 공간
- 스프링 빈 (Spring Bean): 스프링 컨테이너에 의해 생성되고 관리되는, 재사용 가능한 자바 객체 인스턴스

# 2. 관점 지향 프로그래밍

- AOP (Aspect-Oriented Programming): 핵심 비즈니스 로직과 횡단 관심사를 물리적으로 분리하여 모듈화하는 프로그래밍 패러다임
- 프록시 (Proxy): AOP 동작을 위해 원본 타겟 객체를 감싸서 생성된 런타임 가짜 객체로, 타겟 메서드 호출 전후에 제어권을 가로채 부가 기능을 실행 가능
- 포인트컷 (Pointcut): 부가 기능(어드바이스)이 적용될 구체적인 지점(메서드 위치)을 필터링하는 표현식

# 3. 웹 및 네트워크 아키텍처

- 서블릿 (Servlet): 클라이언트의 HTTP 요청을 파싱해 처리하고 응답을 반환하기 위해 톰캣 같은 웹 서버 내부에서 구동되는 자바 표준 기술
- 디스패처 서블릿 (DispatcherServlet): 클라이언트의 HTTP 요청을 중앙에서 단일 진입점으로 받아, 적절한 세부 컨트롤러로 라우팅하는 스프링 MVC의 핵심 프론트 컨트롤러
- REST (Representational State Transfer): HTTP URI로 대상 자원을 명시하고 HTTP 메서드를 통해 해당 자원에 대한 행위(조회, 생성 등)를 정의하는 아키텍처 스타일
- 직렬화 (Serialization): 메모리 상에 존재하는 자바 객체를 네트워크 전송이나 파일 저장에 적합한 데이터 포맷(JSON, 바이트 스트림 등)으로 변환하는 과정
- 역직렬화 (Deserialization): 네트워크를 통해 전달받은 텍스트 포맷(JSON 등)을 다시 메모리 상에서 구동 가능한 자바 객체로 재조립하는 과정
- 리플렉션 (Reflection): 애플리케이션 실행 시점(런타임)에 클래스의 구조(메서드, 필드, 생성자)를 동적으로 분석하고 조작할 수 있는 자바 표준 API
