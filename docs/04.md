# 4. AOP, 관점 지향 프로그래밍을 통한 횡단 관심사의 분리

## 1. 목적

- 애플리케이션에는 크게 두 가지 관심사, 핵심 관심사(Core Concern)와 횡단 관심사(Cross-cutting Concern) 이 존재한다.
- 핵심 관심사는 실제 서비스 기능, 순수 비즈니스 도메인 로직을 말한다.
- 횡단 관심사는 그 외 성능 측정, 트랜잭션 관리, 보안 등 핵심 관심사를 보조할 인프라성 로직을 말한다.
- SRP를 지키기 위해 이 둘을 분리하고 싶지만 OOP 만으로는 해결이 어렵다.
- 스프링은 관점(Aspect)이라는 단위로 프로그램을 설계할 수 있도록 지원한다.

## 2. 구현 내용

```java

@Service
public class MemberServiceImpl implements MemberService {

    private final MemberRepository memberRepository;

    @Autowired
    public MemberServiceImpl(MemberRepository memberRepository) {
        this.memberRepository = memberRepository;
    }

    @Override
    public void join(Member member) {
        long start = System.currentTimeMillis(); // 시간 측정 (횡단 관심사), 4줄
        try {
            // 핵심 비즈니스 로직, 1줄
            memberRepository.save(member);
        } finally {
            long finish = System.currentTimeMillis(); // 시간 측정
            long timeMs = finish - start;
            System.out.println("join 실행 시간: " + timeMs + "ms");
        }
    }
    ...
}
/*
        앞서 작성한 join 기능에 성능 측정을 위해 실행 시간을 계산하는 코드를 도입했다.
        지금은 작은 코드 변경이지만 실제 애플리케이션이 점점 커지게 될 경우
        핵심 비즈니스 로직 코드와 반복되는 횡단 관심사 코드가 섞여 유지보수가 크게 어려워진다.(SRP)
        핵심 비즈니스 로직만 남겨두고 그 외의 코드를 따로 모아 응집성을 높여야 한다.
 */

```

```java

@Aspect
@Component
public class TimeTraceAspect {

    @Around("execution ...")
    public Object execute(ProceedingJoinPoint joinPoint) ...

    {
        // 시간 측정 로직
        try {
            // 비즈니스 로직 수행
            return joinPoint.proceed();
        } ...
    }
    ...
}

/*
        스프링은 AOP 를 위해 시간 측정이라는 관점(Aspect)을 모아 관리하도록 지원한다.
        시간 측정코드를 위한 클래스를 만들고 이를 빈으로 만들어 관리한다.
        이를 적용할 패키지를 선언(Pointcut)하고 해당 패키지의 메서드가 실행되면 메서드 실행 전, 후로 원하는 작업(Advice)을 실행시킬 수 있다.
        
        스프링은 애플리케이션 실행 시점(Runtime)에 대상 객체를 상속받거나 인터페이스를 구현한 동적 프록시를 메모리에 생성해 기존 객체를 대체한다.
        1. `memberService.join()` 호출
        2. 스프링은 실제 객체가 아닌 프록시 객체의 execute() 메서드를 실행시킨다.
        3. execute() 실행 후 먼저 비즈니스 로직 위에 있는 코드가 실행된다.
        4. 실제 클라이언트가 원했던 비즈니스 로직이 호출되며 프록시는 잠시 메서드 실행을 멈춘다.(Suspend)
        5. 비즈니스 로직의 실행이 끝나 리턴되면 멈춰있던 프록시가 다시 깨어나 그 이후 코드가 실행된다.
 */
```
