# 5. Spring MVC 아키텍처, 웹 계층(Web Layer) 도입

## 1. 목적

- Spring Boot를 통해 서버를 도입하고 외부 클라이언트가 요청을 보내 비즈니스 로직을 수행할 수 있도록 한다.
- 프론트 컨트롤러(`DispatcherServlet`) 패턴을 기반으로 Spring MVC 동작 원리를 체험

## 2. 구현 내용

```java
@SpringBootApplication
@EnableAspectJAutoProxy
public class CoreApplication {
    ...
}
/*
        @SpringBootApplication 애노테이션을 통해 내장 톰캣 설정, 컴포넌트 스캔, 자동 설정을 모두 수행한다.
        스프링 부트의 도입으로 이전 스프링에서 작성한 Config 파일들은 필요가 없어졌으므로 애노테이션을 주석처리 한다.
        이후 AOP를 활성화 하기 위해 @EnableAspectJAutoProxy 애노테이션을 여기에 선언한다.
 */
```

```java
@RestController
@RequestMapping("/api/members")
public class MemberController {

    private final MemberService memberService;

    @Autowired
    public MemberController(MemberService memberService) {
        this.memberService = memberService;
    }

    @PostMapping
    public String join(@RequestBody Member member) {
        memberService.join(member);
        return "Member joined successfully: " + member.getName();
    }

    @GetMapping("/{id}")
    public Member findMember(@PathVariable("id") Long id) {
        return memberService.findMember(id);
    }
}
/*
        @RequestMapping 을 통해 이 컨트롤러의 공통 URL 경로를 지정한다.
        
        @PostMapping 을 통해 POST 요청을 처리하는 메서드임을 표시하고,
        클라이언트가 전송한 JSON 데이터를 Member 객체로 변환해 가입을 처리한다.
        @RequestBody 는 클라이언트가 전송한 JSON 텍스트 데이터를 내부의 메시지 컨버터가
        Member 객체로 역직렬화하도록 지시한다.
        
        @GetMapping 을 통해 GET 요청을 처리하는 메서드임을 표시하고,
        URL 경로의 일부를 파라미터로 사용해 비즈니스 로직을 처리한다.
 */
```
